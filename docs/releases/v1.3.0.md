## Scope

v1.3.0 is a **team productivity release** — it adds lightweight artifact management, presentation generation, meeting summaries, status reports, templates, file storage, and download/export capabilities. All features are internal-only, team-scoped, and have no operational control path.

**Core principle**: Help teams produce and organize internal documents without adding governance complexity, approval workflows, or operational mutations.

**No changes to**: approval semantics, Tool Gateway enforcement, Proxmox mutation allowlist, agent autonomy ladder, A5 hard-block, MFA validity window, mutation-window requirements, remediation draft-first model, Python worker isolation, or any operational control path.

## Track Summaries

### Track 1 — Internal Document/Report Workspace
- Migration 033: `artifacts` table (7 artifact types, 3 statuses, file_format CHECK, title NOT NULL)
- Permissions: viewer→read; member→create/read/update; manager+→delete(archive); platform roles→read only
- CRUD: POST/GET/GET-by-id/PATCH/DELETE(archive) `/api/teams/{teamId}/artifacts`
- Team scoping, cross-team 404, source_data sensitive-field redaction, 500KB content_markdown limit
- `ArtifactsPage.tsx` + `ArtifactEditor.tsx`

### Track 2 — Team Presentations via Presenton
- Docker Compose: `presenton` service behind `profiles: ["presenton"]`
- Image pinned: `ghcr.io/presenton/presenton:v0.8.7-beta@sha256:d855169e...`
- Config: PRESENTON_ENABLED, URL, admin credentials, timeout, max_file_bytes — all validated
- `GET /api/teams/{teamId}/artifacts/presenton/status`
- `POST /api/teams/{teamId}/artifacts/generate-presentation`
- Generation: validate → Presenton generate → download → content-type check → max size → SHA256 → upload to MinIO → create storage_object + artifact → audit → return
- S3 nil → 503; upload failure → 500 + no orphan artifact; unreachable → 503
- Presenton container has NO DB/NATS/Redis/MinIO credentials, CAN_CHANGE_KEYS=false, no-new-privileges
- `PresentationModal.tsx`

### Track 3 — Meeting Summary and Action Items
- Migration 034: `artifact_meeting_data` sidecar table (1:1 with artifacts, CASCADE delete)
- JSONB arrays for attendees, agenda_items, decisions, action_items
- Array bounds validation (attendees ≤100, action_items ≤200, duration 0-1440 minutes)
- Recursive sensitive-field sanitization on nested maps, arrays, and key=value patterns
- `POST/GET/GET-by-id/PATCH /api/teams/{teamId}/artifacts/meeting-summaries`
- `MeetingSummaryEditor.tsx`

### Track 4 — Project Status Report Generator
- `POST /api/teams/{teamId}/artifacts/status-reports/generate`
- 7 report sections: summary, milestones, risks, incidents, metrics, asset_actions, remediations
- PostgreSQL-only generation — no Presenton, Python, or Tool Gateway calls
- Date range validation (max 366 days), project team-scoping, section allowlist
- Sensitive operational fields suppressed (no raw incident bodies, action targets, tool params)
- `StatusReportModal.tsx`

### Track 5 — Template Library
- Migration 035: `artifact_templates` table (team_id NULL for system, NOT NULL for custom)
- 6 seeded system templates (Weekly Status, Meeting Summary, Decision Memo, Incident Summary, Architecture Walkthrough, Training Deck)
- Template type CHECK (7 types), metadata JSONB object CHECK, system/team invariant CHECK
- `GET /api/teams/{teamId}/artifact-templates` (system + team, type filter)
- `POST /api/teams/{teamId}/artifact-templates` (team-scoped only, is_system=false enforced)
- `POST /api/teams/{teamId}/artifact-templates/{templateId}/instantiate` (creates artifact from template)
- Cross-team custom template → 404, system templates readable by all teams
- `TemplateGallery.tsx`

### Track 6 — Artifact Storage and Recent Files
- `GET /api/teams/{teamId}/artifacts?include_files=true` — file metadata join (size, format, download_available=false)
- `GET /api/teams/{teamId}/artifacts/recent` — last 20 by updated_at DESC, excludes archived
- `GET /api/teams/{teamId}/artifacts/search?q=` — ILIKE title + description + content_markdown
- `GET /api/teams/{teamId}/artifacts/storage-summary` — total/file/inline counts, file sizes, by_format
- MinIO key pattern verified: `teams/{teamId}/artifacts/{uuid}.{ext}` — team-scoped, UUID-based, no PII
- No bucket names, object keys, or filesystem paths in any response
- Search bar, recent widget, storage summary panel on ArtifactsPage

### Track 7 — Lightweight Sharing/Download Flow
- `GET /api/teams/{teamId}/artifacts/{artifactId}/download` — presigned MinIO URL, 15-min expiry
- `GET /api/teams/{teamId}/artifacts/{artifactId}/export/markdown` — .md download
- `GET /api/teams/{teamId}/artifacts/{artifactId}/export/pdf` — pure Go PDF (no external rendering, no JS)
- Download: file-backed only, archived → 403, inline → 400, cross-team → 404
- Export: content_markdown only, archived → 403, safe sanitized filenames
- Download/Export/Copy Markdown actions in ArtifactEditor
- No public links, no external sharing, no email delivery

### Track 8 — Release Docs, E2E, Regression Validation
- Presenton image pinned by digest (Track 2 release condition closed)
- Release notes (this document)
- v1.3 release checklist
- Security document updates
- Full regression test suite

## Deployment Notes

Migrations 033–035 are applied automatically on API startup. No breaking changes to existing configuration.

```bash
# Standard deployment
git checkout plan/v1.3.0-team-productivity
cd /opt/clarityit
docker compose up -d --build clarityit-api
docker compose restart clarityit-web
sleep 5
docker compose --profile agent up -d

# Optional: Presenton presentation generation
PRESENTON_ENABLED=true \
PRESENTON_ADMIN_PASS=<secure-password> \
docker compose --profile presenton up -d presenton
```

### New Environment Variables

| Variable | Default | Required | Description |
|----------|---------|----------|-------------|
| `PRESENTON_ENABLED` | `false` | No | Enable/disable Presenton integration |
| `PRESENTON_URL` | `http://presenton:80` | No | Presenton service URL |
| `PRESENTON_ADMIN_USER` | `clarityit` | No | Presenton admin username |
| `PRESENTON_ADMIN_PASS` | — | Yes (when enabled) | Presenton admin password (not `changeme`) |
| `PRESENTON_GENERATION_TIMEOUT` | `120s` | No | Generation request timeout (10-600s) |
| `PRESENTON_MAX_FILE_BYTES` | `52428800` (50MB) | No | Max file size (1MB-100MB) |
