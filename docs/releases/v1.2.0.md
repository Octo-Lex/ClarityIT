## Scope

v1.2.0 is an **operational intelligence release** — it improves decision quality, confidence, explainability, and learning loops **without expanding autonomous execution authority**. All seven tracks are advisory, read-only, or evaluation-only. No new mutation classes, no A5 autonomy, no autonomous remediation expansion.

**Core principle**: Judge better, explain more, compare safely, learn from outcomes — but do not act without human approval.

**No changes to**: approval semantics, Tool Gateway enforcement, Proxmox mutation allowlist, agent autonomy ladder, A5 hard-block, MFA validity window, mutation-window requirements, or the remediation draft-first model.

## Track Summaries

### Track 1 — Recommendation Evidence Packs
- Migration 029: `recommendation_evidence` table
- `GET /api/teams/{teamId}/recommendations/{recommendationId}/evidence`
- Confidence score (0.0–1.0) + level (low/medium/high)
- Supporting evidence, conflicting evidence, risk notes, missing info
- `stale_after` marks stale evidence (visible, not deleted)
- Evidence persisted by Go control plane — Python worker proposes, Go validates
- Evidence sanitization: sensitive patterns stripped from text + map keys
- `EvidencePanel.tsx` in RemediationPanel

### Track 2 — Incident Pattern Detection
- `GET /api/teams/{teamId}/incidents/patterns` (read-only, computed from PostgreSQL)
- Query params: `window_days` (1–90), `min_occurrences` (2–20)
- 4 pattern types: `recurring_asset`, `recurring_symptom`, `cluster`, `noisy_asset`
- Stable pattern IDs via SHA256, confidence bounded 0.0–1.0
- No raw incident bodies in response; symptom descriptions sanitized
- `PatternCards.tsx` on Incidents list page with advisory badges

### Track 3 — Approval Policy Simulation
- `POST /api/admin/approval-policy/simulate` (platform_owner gated)
- Draft policy (low/medium/high/critical) + scenarios array (1–50)
- Returns per-scenario simulation results + policy diff vs current
- `simulation_only=true`, `live_policy_changed=false` always
- No live policy mutation, no approval request/decision creation, no Tool Gateway call, no audit/outbox emission
- `PolicySimulationPanel.tsx` in AdminApprovals page

### Track 4 — Change-Risk Scoring
- `GET /api/teams/{teamId}/assets/{assetId}/risk-score?action={action}`
- Supported actions: `proxmox.start`, `proxmox.shutdown`, `proxmox.stop`, `proxmox.snapshot`
- 7 scoring factors: action_type, asset_criticality, recent_incidents, blast_radius, time_of_day, mutation_window_status, past_outcomes
- Score bounded 0–100, levels: low (0–24), medium (25–49), high (50–79), critical (80–100)
- Advisory only — does not bypass approval/MFA/policy/mutation-window
- Dry-run integration: `risk_score` object in preview response
- `RiskScoreDisplay` component in AssetActions preview

### Track 5 — Post-Action Outcome Tracking
- Migration 030: `action_outcomes` table
- `POST/GET /api/teams/{teamId}/asset-actions/{actionId}/outcome`
- `POST/GET /api/teams/{teamId}/remediations/{proposalId}/outcome`
- Terminal status required (succeeded/failed/cancelled)
- Upsert behavior (one outcome per action/proposal)
- Sensitive field sanitization (key=value regex patterns)
- Feedback signals: `past_outcomes` factor in risk scoring, `repeated_failed_outcomes` pattern type
- `OutcomePanel.tsx` shared component
- No retry/follow-up execute buttons

### Track 6 — Context Graph Quality Controls
- Migration 031: `context_relation_reviews` sidecar table
- `GET /api/teams/{teamId}/context/quality`
- `POST /api/teams/{teamId}/context/relations/{relationId}/confirm`
- `POST /api/teams/{teamId}/context/relations/{relationId}/dismiss`
- Quality signals: stale nodes (30-day default), low-confidence relations (< 0.60 default), conflicting relations
- Quality score 0–100 (penalties for stale/low-conf/conflicts, bonus for confirmed)
- Advisory only — confirm/dismiss does not delete or rewrite graph data
- No Python worker call, no Tool Gateway call
- `ContextQualityPanel.tsx` component

### Track 7 — Agent Recommendation Evaluation Harness
- Migration 032: `agent_evaluation_runs` + `agent_evaluation_scenario_results`
- `POST /api/admin/agent-evaluation/run` (platform_owner)
- `GET /api/admin/agent-evaluation/results` (latest run)
- `GET /api/admin/agent-evaluation/runs/{runId}` (detail)
- 5 golden scenarios (deterministic fixtures)
- 4 scoring dimensions: correctness, safety, explainability, quality (each 0.0–1.0)
- Evaluation mode only — no Tool Gateway, no Proxmox, no approvals, no asset_actions, no remediation_proposals, no context graph mutations
- Sensitive fields redacted, chain_of_thought never persisted
- `AgentEvaluationPanel.tsx` component

### Track 8 — Release Docs, Regression, and Operational Validation
- Release notes (this document)
- v1.2 release checklist
- Security document updates
- Full regression test suite
- Deployment verification
