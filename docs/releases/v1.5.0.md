# ClarityIT v1.5.0 — Knowledge Productivity

**Release Date**: 2026-06-17
**Branch**: `plan/v1.5.0-knowledge-productivity`
**Tag**: `v1.5.0-knowledge-productivity`
**Migration Range**: 039–040

## Overview

v1.5.0 turns ClarityIT's accumulated documents, artifacts, incidents, context graph, and operational data into searchable, explainable internal knowledge. Users can search across all team content, see related knowledge in context, ask source-grounded questions, organize findings into collections, and monitor knowledge quality — all powered by PostgreSQL full-text search, with no external search engine or vector database.

**PostgreSQL FTS only. No external search SaaS. No vector DB. No public sharing.**

## Tracks

### Track 1 — Knowledge Index Foundation
- `knowledge_items` table with auto-populating `search_vector` tsvector trigger
- `knowledge_chunks` table for sub-item content segmentation
- GIN indexes for fast full-text search
- Weight A (title), B (summary), C (content_text) ranking
- 13 source types: artifact, clarity_document, meeting_summary, status_report, presentation, template, work_item, incident, project, asset, remediation, approval, context_node
- 6 permissions: knowledge.search, knowledge.read, knowledge.create, knowledge.update, knowledge.delete, knowledge.ask
- Team-scoped: UNIQUE (team_id, source_type, source_id)
- Migration 039

### Track 2 — Source Indexers
- `indexer.go`: IndexSource (upsert + chunk replacement), ChunkContent (paragraph-based, MaxChunkSize=4000, MaxChunks=100)
- `sources.go`: 12 of 13 source extractors (project deferred — no project table)
- `sanitizer.go`: Regex stripping of secrets/tokens/prompts/COT, metadata key removal, SHA256 hashing
- `reindex.go`: ReindexAll orchestrator, admin HTTP handlers
- 9 source-trigger indexing hooks: artifact Create/Patch, document Create/Patch/Generate, template Create/Instantiate, meeting Create, status report Generate
- Content-hash comparison skips unchanged items

### Track 3 — Unified Search UI
- `/knowledge` route with KnowledgeSearchPage
- Source-type filtering (14 filter buttons)
- Snippet highlighting via ts_headline `<mark>` tags
- Result routing: getSourceRoute() maps source_type → contextual route
- States: empty, loading, error (safe message), no-results, results
- No Ask/Q&A, no share/approval/execute controls

### Track 4 — Related Knowledge Panels
- `GET /api/teams/{teamId}/knowledge/related` endpoint
- 5 deterministic ranking signals: explicit_link (0.90), context_edge (0.75-0.80), content_similarity (0.5×ts_rank), same_source_family (0.30), recent_related (0.10)
- Candidates deduplicated by (source_type|source_id), first reason wins
- Panel placements: DocumentEditorPage (clarity_document), ArtifactsPage (artifact), IncidentDetail (incident)
- Collapsible panel with loading/empty/error states

### Track 5 — Ask Clarity Knowledge Q&A
- `POST /api/teams/{teamId}/knowledge/ask` endpoint
- FTS retrieval from `knowledge_chunks` → Python worker `/knowledge-ask`
- Source-grounded answers with citations validated against retrieved set
- Confidence levels: low/medium/high
- Safe degradation on worker failure (returns sources, low confidence)
- No-sources path skips Python entirely
- Forbidden field validation: chain_of_thought, thinking, internal_reasoning, tool_calls, action, mutation, execute
- No audit/outbox/persistence writes
- Worker timeout: 60s, response cap: 100KB

### Track 6 — Knowledge Collections and Saved Findings
- `knowledge_collections` table with soft-delete (archived_at)
- `knowledge_collection_items` table (team-scoped, cascade delete)
- `saved_knowledge_answers` table (collection_id nullable, ON DELETE SET NULL)
- Duplicate item handling: ON CONFLICT DO NOTHING, returns 200 with duplicate=true
- Save integrations: SearchResultCard, RelatedKnowledgeCard, AskClarityAnswer, AskClaritySourceCard
- Collection detail with item management (add/remove/edit)
- Saved answers list + detail with preserved source cards
- No audit/outbox writes (internal org feature)
- Migration 040

### Track 7 — Knowledge Quality Controls
- Quality dashboard: Total, Stale, Duplicates, Orphans summary cards
- Stale detection: `stale_after < NOW()` (90-day freshness window)
- Duplicate detection: same content_hash, count > 1
- Orphan detection: source artifact deleted but knowledge item remains
- All endpoints read-only, team-scoped, no mutation
- By-source-type breakdown
- No migration needed (uses existing columns)

### Track 8 — Release Docs, E2E, Regression
- Release notes (this document)
- Release checklist
- E2E smoke tests for v1.5.0 knowledge features
- Full regression verification

## Python Worker

- New endpoint: `/knowledge-ask` on port 9100 (internal-only)
- Receives question + bounded snippets only
- No DB/MinIO/NATS/Redis access
- `_sanitize_response` strips 9 forbidden fields
- `_extract_citations` finds source_keys in answer text
- `_assess_confidence`: low/medium/high based on answer length + citations

## Security Boundaries

- PostgreSQL FTS only — no external search engine, no vector database
- Knowledge index is team-scoped — all queries filtered by team_id
- search_vector auto-populated by trigger — never user-controlled
- Source-trigger indexing fires asynchronously — no blocking
- Ask Clarity: no audit/outbox/persistence, no operational side effects
- Collections: no audit/outbox writes, internal org feature only
- Quality: read-only, no mutation
- No public search, no external sharing, no publishing
- No Tool Gateway expansion, no Proxmox changes
- No A5 autonomy, no agent execution expansion
