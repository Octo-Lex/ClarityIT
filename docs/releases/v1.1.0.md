## Scope

v1.1.0 is a **refinement release** — it does not expand the platform architecture or introduce new operational capabilities. It removes carried-forward security debt (Track 1), adds operational confidence tools (Tracks 2–4, 6–7), adds an optional MFA factor (Track 5), and provides final release evidence (Track 8).

**No changes to**: approval semantics, Tool Gateway enforcement, Proxmox mutation allowlist, agent autonomy ladder, or the A5 hard-block.

## Track Summaries

### Track 1 — Resolve Dev-Only Audit Exception
- Upgraded Vite 4→8 (Rolldown/Oxc replaces esbuild), Vitest→4.1.8, @vitejs/plugin-react→6
- `npm audit --audit-level=high` now reports **0 vulnerabilities** (was 6 high)
- Risk acceptance document closed

### Track 2 — Proxmox Mutation Change-Window Workflow
- Migration 026: `proxmox_mutation_windows` table
- `PROXMOX_MUTATION_ENABLED=true` is necessary but not sufficient — active mutation window also required
- POST/GET `/api/admin/proxmox/mutation-window`, POST `/{id}/close`
- Duration bounded 1–60 min, reason required, one active window, auto-expiry
- 15 mutation window tests

### Track 3 — Backup Verification Reporting in UI
- `GET /api/admin/backup-status` — scans backup directory for PostgreSQL/MinIO files
- Age status: green (≤24h), yellow (≤72h), red (>72h), missing
- Path redaction: basename only, no raw filesystem paths
- Frontend: Backup Status card in AdminOps
- Docker volume mount: host `/opt/clarityit/backups` → container `/var/lib/clarityit/backups:ro`

### Track 4 — Approval Escalation and Expiry Notifications
- Migration 027: `expiring_notified_at` column on `approval_requests`
- Background monitor goroutine: detects expiring/expired approvals
- Expiring: one-time notification via `expiring_notified_at` (no-repeat)
- Expired: transitions pending→expired, writes audit + outbox
- Frontend: expiry badges (green/yellow/red), time remaining, status filter tabs, WS toast notifications
- Config: `APPROVAL_MONITOR_ENABLED`, `APPROVAL_MONITOR_INTERVAL_SECONDS`, `APPROVAL_EXPIRING_THRESHOLD_PERCENT`

### Track 5 — WebAuthn/FIDO2 MFA Option
- Migration 028: `user_webauthn_credentials` table
- 6 endpoints: register start/finish, authenticate start/finish, list credentials, disable credential
- go-webauthn/webauthn v0.17.4 library
- **MFA step-up only** — refreshes `recent_mfa_at` after authenticated session
- Production requires HTTPS origin; dev allows localhost http
- Disabled by default (`WEBAUTHN_ENABLED=false`)
- Audit/outbox events with credential_id redaction
- 15 backend + 7 frontend tests

### Track 6 — Proxmox Mutation Dry-Run Previews
- `dry_run=true` on action creation returns preview without side effects
- Validates asset ownership, provider, metadata, action type, snapshot name
- Computes risk/approval/MFA/min-approvers, reports mutation-window and feature-flag state
- No asset_actions row, no approval_request, no Proxmox API call
- Optional `asset.action.previewed` audit event (sanitized, `dry_run:true`)
- Frontend: Preview button on each action card with no-action warning
- 15 backend + 7 frontend tests

### Track 7 — Operational Metrics
- `GET /api/admin/metrics` (platform_owner gated)
- Approvals: counts by status + avg time to decision
- Remediations: counts by status
- Asset Actions: counts by status/type + success rate (succeeded+failed denominator)
- Agents: run counts + avg reasoning time
- Computed at request time from PostgreSQL — no Redis cache, no charting library
- Frontend: `/admin/metrics` page with summary cards
- 10 backend + 7 frontend tests

## Configuration (v1.1 additions)

| Variable | Default | Purpose |
|----------|---------|---------|
| `APPROVAL_MONITOR_ENABLED` | `true` (prod) | Enable approval expiry monitor |
| `APPROVAL_MONITOR_INTERVAL_SECONDS` | `60` | Monitor check interval (clamped 5–3600) |
| `APPROVAL_EXPIRING_THRESHOLD_PERCENT` | `25` | Threshold for expiring notification (clamped 1–90) |
| `WEBAUTHN_ENABLED` | `false` | Enable WebAuthn/FIDO2 MFA option |
| `WEBAUTHN_RP_ID` | _(empty)_ | WebAuthn Relying Party ID |
| `WEBAUTHN_RP_ORIGIN` | _(empty)_ | WebAuthn Relying Party origin (HTTPS required in prod) |
| `WEBAUTHN_RP_DISPLAY_NAME` | `ClarityIT` | WebAuthn display name |
| `BACKUP_DIR` | `/var/lib/clarityit/backups` | Backup directory for status checks |
